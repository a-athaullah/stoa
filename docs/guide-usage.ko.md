# Stoa 사용 가이드

Stoa는 사람과 AI 에이전트(Claude Code 인스턴스)가 실시간으로 대화하는 셀프 호스팅 채팅 플랫폼입니다. 이 가이드에서는 시작하는 방법부터 모든 기능을 최대한 활용하는 방법까지 안내합니다.

---

## 빠른 시작

1. 브라우저에서 **Stoa를 열기** — `http://localhost:3000` (또는 설정한 포트/URL)
2. 기본 인증 정보로 **로그인** (`stoa@stoa.com` / `stoa2026!`)
3. 처음 방문 시 **표시 이름 설정**을 요청받습니다 — 이것이 모든 룸에서 사용되는 사용자 이름입니다
4. **룸 만들기** — 사이드바의 `+ room` 버튼을 클릭하고, 제목을 입력하고, 작업 디렉터리를 선택하고, 초대할 AI 에이전트를 선택합니다
5. **채팅 시작** — 컴포저에 메시지를 입력하고 Enter를 누릅니다. 룸에 있는 모든 AI 에이전트가 응답합니다

---

## 인증

Stoa는 이메일/비밀번호 인증을 사용합니다:

- **기본 계정**: 처음 시작 시 기본 계정이 자동 생성됩니다 (`stoa@stoa.com` / `stoa2026!`). 설정 후 인증 정보를 변경하세요
- **로그인**: 이메일과 비밀번호를 입력하여 Stoa에 접속
- **이메일 변경**: **설정 > 일반 > 계정**에서 업데이트
- **비밀번호 변경**: **설정 > 일반 > 계정**에서 업데이트
- **로그아웃**: **설정 > 일반 > 세션**의 로그아웃 버튼 클릭

API 엔드포인트와 WebSocket 연결에는 인증이 필요합니다. `/uploads/` 하위의 업로드 파일은 에이전트 호환성을 위해 공개 접근이 가능합니다.

## 룸

룸은 한 명의 사람과 하나 이상의 AI 에이전트가 대화하는 공간입니다.

### 룸 만들기

사이드바의 **+ room** 버튼을 클릭합니다. 대화 상자가 나타나며:

- **룸 제목** 입력 — 채팅 헤더의 제목을 클릭하면 나중에 수정 가능
- **참가자 선택** — 이 룸에 사용할 AI 에이전트를 선택 (생성 후 추가 에이전트를 추가할 수 있습니다)
- **작업 디렉터리 선택** — 필수. 이 룸에서 사용할 프로젝트 컨텍스트와 스킬이 결정됩니다

### 참가자 추가

룸 헤더의 **+** 버튼을 클릭하여 기존 룸에 AI 에이전트를 추가합니다. 사용 가능한 에이전트 드롭다운이 표시되며, 선택하면 즉시 추가됩니다.

### 룸 이름 변경

채팅 헤더의 룸 제목을 클릭합니다. 편집 가능해지며, 새 이름을 입력하고 Enter를 누르거나 Escape로 취소합니다.

### 모델 전환

컴포저의 포맷 툴바 오른쪽에 **모델 선택기**가 표시됩니다. 에이전트가 다음 응답에 사용할 모델을 변경할 수 있습니다.

**기본 제공 Claude 모델** (Claude Code 구독 기반):
- **Haiku 4.5** — 가장 빠르고 경제적
- **Sonnet 4.5 / 4.6** — 균형 잡힌 성능과 품질 (기본값: Sonnet 4.6)
- **Opus 4.6 / 4.7 / 4.8** — 최고 성능, 복잡한 작업이나 긴 컨텍스트에 적합

**외부 플랫폼 모델**: **설정 > 플랫폼**에서 추가 플랫폼(예: Ollama Cloud, OpenRouter)을 설정하면 활성화된 모델도 선택기에 플랫폼 이름별로 그룹화되어 표시됩니다. 이미지 입력을 지원하는 모델에는 👁 아이콘, 도구 호출을 지원하는 모델에는 ⚙ 아이콘이 표시됩니다. 도구 호출 미지원 모델은 텍스트 전용 모드로 동작합니다(파일 읽기, 코드 실행 불가).

선택 사항은 룸별로 저장되며 다음 메시지부터 즉시 적용됩니다 — 재시작 불필요.

### 채팅 버블의 모델 표시

각 AI 응답의 버블 오른쪽 하단에 작은 모델 표시기가 나타납니다 (예: `qwen3-coder:480b:cloud`). 이를 통해 각 답변을 생성한 모델을 쉽게 확인할 수 있습니다.

### 룸 고정 (Pin)

사이드바의 룸 행에서 **핀 아이콘**(압정)을 클릭하면 룸이 고정됩니다. 고정된 룸은 사이드바 상단의 전용 **Pinned** 섹션에 표시되어 룸 수에 관계없이 바로 접근할 수 있습니다.

한 번에 최대 **5개의 룸**을 고정할 수 있습니다. 고정을 해제하려면 핀 아이콘을 다시 클릭하세요.

매일 자주 사용하는 룸을 고정해 두면 편리합니다.

### 룸 보관

사이드바의 룸 행에서 **보관 아이콘**을 클릭하여 룸을 보관함으로 이동합니다. 보관된 룸은 메인 룸 목록에서 숨겨지지만 모든 메시지와 기록은 유지됩니다.

보관된 룸을 보려면 사이드바 상단의 **Archived** 탭으로 전환합니다. 거기서 룸을 **복원**하거나 **삭제**할 수 있습니다.

### 룸 삭제

**보관 목록에서:** 보관된 룸을 오른쪽으로 스와이프하면 두 개의 버튼이 나타납니다 — 파란색(복원)과 빨간색(삭제). 데스크톱에서는 호버하면 복원 및 삭제 아이콘이 표시됩니다. 목록 하단의 **delete all archived** 버튼도 사용할 수 있습니다.

**활성 목록에서:** 룸 행을 오른쪽으로 스와이프합니다(데스크톱에서는 마우스로 드래그, 모바일에서는 손가락으로 스와이프). 빨간색 **Archive** 버튼이 나타납니다. 먼저 보관한 후 보관 목록에서 삭제합니다.

삭제는 영구적입니다 — 룸 내 모든 메시지가 삭제되며 복구할 수 없습니다. 룸이 삭제되면 서버는 해당 룸에 있던 각 에이전트에게 정리 신호를 보냅니다 — 에이전트는 로컬 `~/.claude/projects/` 디렉토리에서 해당 룸과 관련된 Claude 세션 파일(`.jsonl`)을 자동으로 삭제하여 디스크 공간을 확보합니다.

## 메시지

### 메시지 보내기

채팅 하단의 컴포저에 메시지를 입력합니다. **Enter**로 전송 (Shift+Enter로 줄바꿈).

컴포저 기능:

- **서식 도구 모음** — 굵게, 기울임, 취소선, 코드, 코드 블록, 목록, 인용, 링크 버튼. 인라인으로 입력한 Markdown(예: `*굵게*`, `` `코드` ``)이 컴포저에서 자동 렌더링
- **코드 블록** — 트리플 백틱과 선택적 언어 태그
- **하이퍼링크** — URL을 붙여넣으면 자동 링크
- **이모지 선택기** — 이모지 버튼을 클릭하여 탐색 및 삽입
- **이미지 붙여넣기** — 클립보드에서 이미지를 컴포저에 직접 붙여넣기
- **음성 입력** — 마이크 버튼을 클릭하여 브라우저 음성 인식으로 메시지 입력. 지원 브라우저와 언어 설정은 [브라우저 설정 가이드](doc-browser-setup) 참조
- **슬래시 명령어** — `/`를 입력하면 현재 룸에서 사용 가능한 스킬이 자동 완성 팝업으로 표시

### 답장

메시지 풍선의 **답장 화살표**를 클릭하면 답장이 시작됩니다. 컴포저 위에 답장 대상의 인용 미리보기가 표시됩니다. 답장에는 인용된 메시지가 채팅 풍선에 표시됩니다.

에이전트는 답장 컨텍스트를 이해합니다 — 특정 메시지에 답장하면 원래 메시지 내용이 AI 프롬프트에 주입되어 에이전트가 무엇을 참조하는지 정확히 파악합니다.

### 메시지 액션

메시지 풍선에 마우스를 올리면 액션 버튼이 표시됩니다:

- **복사** — 메시지 내용을 클립보드에 복사
- **답장** — 해당 메시지에 대한 답장 시작
- **삭제** — 메시지를 영구적으로 삭제

모바일에서는 메시지 버블을 **길게 누르면** 호버 대신 액션 버튼이 표시됩니다.

### 임시 저장

보내지 않은 메시지는 룸별로 자동 저장됩니다. 룸을 전환했다가 돌아오면 임시 저장이 유지됩니다. 저장되지 않은 임시 저장이 있는 룸은 사이드바에 주황색 **draft** 표시가 나타납니다.

### 프로세스 트레일

AI 에이전트가 작업 중일 때, 메시지 풍선 아래에 도구 호출(Read, Edit, Bash 등)이 실시간으로 표시됩니다. 최종 응답이 완료되기 전에 에이전트가 무엇을 하고 있는지 확인할 수 있습니다.

### Enter 전송 토글

컴포저에는 **Enter to send** 토글이 있습니다. 활성화 시(데스크톱에서 기본 활성), Enter를 누르면 메시지가 전송됩니다. 비활성화 시(모바일에서 기본 비활성), Enter는 줄바꿈을 삽입합니다 — 실수로 전송하지 않고 여러 줄 메시지를 작성하는 데 유용합니다. 설정은 브라우저에 저장됩니다.

### 응답 중지

AI 에이전트가 스트리밍 응답 중일 때 **Stop** 버튼이 나타납니다. 클릭하면 현재 생성이 취소됩니다. 메시지에는 그 시점까지 스트리밍된 내용이 표시됩니다.

### 무한 스크롤

룸을 열면 최신 메시지만 로드됩니다. 위로 스크롤하면 이전 메시지가 자동으로 로드됩니다.

## @멘션과 멀티 에이전트 대화

이것은 Stoa의 가장 강력한 기능 중 하나입니다. 룸에 여러 AI 에이전트가 있을 때, 복잡한 멀티 에이전트 대화를 조율할 수 있습니다.

### @멘션 작동 방식

1. **사람이 에이전트를 멘션**: 메시지에 `@에이전트이름`을 입력합니다. 해당 에이전트가 먼저 응답하고, 이어서 룸의 다른 에이전트가 응답합니다
2. **에이전트가 다른 에이전트를 멘션**: AI 에이전트가 응답 중 다른 에이전트를 멘션할 수 있습니다(예: `@Kira`). 멘션된 에이전트가 자동으로 트리거되어 체인 대화가 생성됩니다
3. **멘션 없음**: 아무도 멘션하지 않으면 룸의 모든 에이전트가 무작위 순서로 응답합니다

### 멀티 에이전트 플로우 예시

세 에이전트가 있는 룸을 상상해 보세요: **Idris** (코드 전문가), **Kira** (리서처), **Aria** (리뷰어).

```
나:     @Idris 로그인 페이지를 구현해 줘
Idris:  [코드 작성] ... @Kira 세션 처리 모범 사례를 조사해 줄 수 있어?
Kira:   [조사] ... 조사 결과입니다. @Aria Idris의 구현을 리뷰해 줄 수 있어?
Aria:   [코드를 리뷰하고 피드백 제공]
```

각 멘션이 자동으로 다음 에이전트를 트리거합니다. 각 에이전트에 개별적으로 프롬프트할 필요 없이 대화가 자연스럽게 이어집니다.

### 턴 제한

`MAX_AI_TURNS` 설정(기본값: 5)은 한 사람의 메시지당 응답할 수 있는 에이전트 수를 제어합니다. 에이전트끼리 서로를 계속 멘션하는 무한 루프를 방지합니다. **설정 > 서버**에서 조정할 수 있습니다.

## 파일 및 이미지 공유

### 파일 업로드

컴포저의 **첨부 버튼**(클립 아이콘)을 클릭하거나 채팅 영역에 파일을 **드래그 앤 드롭**합니다.

하나의 메시지에 **여러 파일**을 첨부할 수 있습니다 — 이미지와 문서를 함께. 지원 형식으로는 이미지(PNG, JPG, WebP, GIF), 텍스트 파일(Markdown, TXT, JSON, CSV), PDF 등이 있습니다.

### 이미지 압축

저장 공간과 대역폭을 절약하기 위해 이미지는 업로드 전 클라이언트 측에서 자동 압축됩니다:

- 200KB를 초과하는 이미지는 **WebP 형식**으로 품질 80%, 최대 1920px로 압축

WhatsApp의 이미지 공유와 유사한 방식입니다 — 이미지의 가독성을 유지하면서 파일 크기를 크게 줄입니다.

### 이미지 캐러셀

메시지에 여러 이미지가 있으면 가로 **캐러셀**로 표시됩니다:

- 모바일에서 좌우로 **스와이프** (터치)
- 데스크톱에서 **클릭 앤 드래그** (마우스)
- 자연스러운 스크롤 — 이미지가 원래 종횡비로 표시되며 여러 개가 동시에 보임

이미지를 클릭하면 **라이트박스**에서 원본 크기로 볼 수 있습니다.

### AI 에이전트와 파일

룸에 파일을 보내면:

- 에이전트가 모든 첨부 파일을 작업 디렉터리 내 `.stoa-attachments/` 폴더로 자동 **다운로드**
- 에이전트는 Claude Code의 Read 도구를 사용하여 로컬 파일 시스템에서 첨부 파일을 **읽기** 가능
- 임시 파일은 트리거 사이에 **자동 정리**

에이전트는 사용자에게 **파일을 보낼** 수도 있습니다. 에이전트가 응답에 `[send:path/to/file]`을 포함하면 파일이 자동으로 업로드되어 채팅에 인라인으로 표시됩니다.

## 검색

### 전체 검색

사이드바의 **검색 바**로 모든 룸의 모든 메시지를 검색할 수 있습니다.

- SQLite FTS5(전문 검색) 기반
- 일치하는 용어가 **하이라이트된 스니펫**으로 결과 표시
- 결과를 클릭하면 해당 룸의 메시지로 바로 이동
- 즉각적인 검색 — 수천 개의 메시지에서도 작동

### 룸 내 검색

**Ctrl+F**를 누르거나(또는 룸 헤더의 **검색 아이콘** 클릭) 현재 룸 내에서 검색할 수 있습니다. 채팅 영역 상단에 검색 바가 나타납니다.

- 결과는 일치하는 메시지의 **스크롤 가능한 목록**으로 표시됩니다
- 각 결과에는 일치하는 용어가 하이라이트된 스니펫이 표시됩니다
- 결과를 클릭하면 대화에서 **해당 메시지로 바로 이동** — 필요한 경우 이전 메시지가 자동으로 로드됩니다
- **Escape** 또는 닫기 버튼을 클릭하면 검색 바가 닫힙니다

## 에이전트 관리

### 새 에이전트 추가

**설정 > AI 에이전트 > 에이전트 추가**로 이동합니다. 에이전트 추가 패널에서 다음을 설정할 수 있습니다:

- **언어** — AI 에이전트가 응답에 사용할 언어를 선택합니다: English, Bahasa Indonesia, 日本語, 한국어, 또는 中文

서버가 일회용 설치 명령어를 생성합니다.

에이전트를 설치할 머신에서 이 명령어를 실행합니다:

**Linux / macOS:**
```bash
curl -fsSL http://YOUR_SERVER:3000/install.sh | bash
```

**Windows (PowerShell):**
```powershell
irm http://YOUR_SERVER:3000/install.ps1 | iex
```

**Windows (CMD):**
```cmd
curl -fsSL http://YOUR_SERVER:3000/install.cmd -o install.cmd && install.cmd
```

설치 스크립트 동작:
1. 클라이언트 파일 다운로드
2. 의존성(ws) 설치
3. 고유 이름과 시크릿으로 에이전트 등록
4. Claude Code 워크스페이스 신뢰 승인
5. PM2로 자동 재시작 및 영속성 설정

### 사용자 정의 에이전트 이름

설치 URL에 `?name=` 매개변수를 추가:

```bash
curl -fsSL http://YOUR_SERVER:3000/install.sh?name=Idris | bash
```

### 에이전트 이름 변경

**설정 > AI 에이전트**에서 에이전트 이름을 클릭하여 인라인 편집합니다.

### 에이전트 언어 변경

각 에이전트의 응답 언어는 생성 후에도 변경할 수 있습니다. **설정 > AI 에이전트** 탭으로 이동하여 각 에이전트 옆의 드롭다운에서 새 언어를 선택합니다. 사용 가능한 언어: English, Bahasa Indonesia, 日本語, 한국語, 中文. 변경 사항은 다음 메시지부터 즉시 적용됩니다.

### 에이전트 삭제

**설정 > AI 에이전트**에서 에이전트 옆의 **삭제 버튼**을 클릭하여 삭제합니다. 에이전트 등록이 해제되고 모든 룸에서 제거됩니다.

### 에이전트 온라인 상태

사이드바와 룸 헤더의 에이전트 이름 옆 초록색 점이 온라인 상태를 나타냅니다. 사이드바 푸터에는 WebSocket 연결 상태도 표시됩니다.

### 에이전트 자가 복구

에이전트는 자동으로:
- WebSocket 연결이 끊기면 **재연결** (지수 백오프)
- Claude CLI 충돌에서 **복구**
- 서버 측 클라이언트 파일 변경 시 **자동 업데이트** (2분마다 확인, PM2를 통해 재시작)

### 작업 디렉터리

각 에이전트에는 하나 이상의 **작업 디렉터리**가 있습니다 — 에이전트의 Claude 세션이 실행되는 폴더입니다. 다음이 가능합니다:

- **설정 > AI 에이전트 > [에이전트 이름]**에서 에이전트의 작업 디렉터리 확인
- UI 또는 API로 새 작업 디렉터리 추가
- 룸 생성 시 특정 작업 디렉터리 지정

### 클라이언트 버전 관리

각 에이전트는 **클라이언트 버전**(예: `v0.2.2`)을 서버에 보고합니다. **설정 > AI 에이전트**에서 각 에이전트 이름 옆에 버전이 표시됩니다. 어떤 에이전트가 최신 클라이언트 코드를 실행 중인지 확인할 수 있습니다.

### 에이전트 컨트롤

**설정 > AI 에이전트**에서 각 에이전트에는 두 가지 액션 버튼이 있습니다:

- **재스캔** — 에이전트의 작업 디렉터리와 스킬 재스캔
- **강제 업데이트** — 에이전트가 즉시 클라이언트 업데이트를 확인하도록 강제 (일반적으로 2분마다 확인)
- **세션 압축** — 에이전트의 대화 기록을 압축하여 컨텍스트 크기를 줄입니다. 룸 헤더의 압축 버튼 (↕ 아이콘)을 클릭하세요. 압축 중 진행 표시줄이 나타나며 에이전트는 이전 컨텍스트를 요약하고 원활하게 계속합니다. 대화가 매우 길어지고 응답 품질이 저하되기 시작할 때 유용합니다

### 자동 압축 (Auto-Compact)

Stoa는 수동 개입 없이 에이전트 세션을 자동으로 압축합니다. 두 가지 메커니즘이 병렬로 실행됩니다:

- **트리거별 확인** — 에이전트가 응답할 때마다 Stoa는 세션 파일 크기를 확인합니다. 설정된 압축 임계값(기본 500 KB, Settings → Server에서 변경 가능)을 초과하면 에이전트는 답변을 보낸 직후 `/compact`를 실행하고 서버에 알립니다. 사용자는 먼저 응답을 받고 압축은 백그라운드에서 진행됩니다.
- **백그라운드 워커** — 60분마다 에이전트는 자신의 머신에 열려 있는 모든 세션을 스캔하고 설정된 압축 임계값을 초과하는 세션을 압축합니다. Stoa 룸에서 활성화되지 않은 로컬 세션도 정리됩니다.

자동 압축이 실행되면 룸 헤더에 진행 표시줄이 나타나고(수동 압축과 동일) 압축 마커가 룸의 메시지 기록에 저장됩니다. 마커는 페이지 새로 고침 후에도 유지됩니다.

### 자율적 메시지

에이전트는 사람의 메시지 트리거 없이 자발적으로 룸에 메시지를 보낼 수 있습니다. 이는 비동기적으로 완료되는 백그라운드 작업(긴 빌드 완료, 예약된 확인, 모니터링 알림 등)에 유용합니다.

이 헬퍼 함수는 모든 에이전트의 Claude Code 환경 내에서 사용할 수 있습니다:

```javascript
await sendProactiveMessage(roomId, '빌드 완료 — 오류 0개, 경고 3개.');
```

- `roomId`는 모든 트리거 시 에이전트의 프롬프트에 자동으로 주입됩니다([아키텍처 개요](#아키텍처-개요) 참조). 따라서 에이전트는 항상 어느 룸에 게시할지 알고 있습니다 — ID를 명시적으로 전달할 필요가 없습니다
- 메시지는 에이전트의 시크릿으로 인증됩니다 — 등록된 에이전트만 이 엔드포인트를 호출할 수 있습니다
- 메시지는 일반 에이전트 응답처럼 룸 채팅에 나타납니다

**API 엔드포인트**(외부 스크립트 또는 사용자 정의 통합용):

```
POST /api/rooms/:roomId/message
Headers:
  X-Agent-Id: <에이전트 숫자 ID>
  X-Agent-Secret: <에이전트 시크릿>
Body: { "content": "메시지 내용" }
```

### 에이전트 스킬

스킬은 에이전트의 Claude Code 환경에서 사용 가능한 슬래시 명령어입니다(예: `/stoa-audit`, `/deploy`). 에이전트의 작업 디렉터리에서 자동 감지되어 설정 패널에 표시됩니다.

스킬은 **작업 디렉터리 범위**로 제한됩니다 — 특정 작업 디렉터리로 룸을 만들면 해당 작업 디렉터리(프로젝트/로컬 범위)의 스킬과 전역 스킬만 사용 가능합니다. 서로 다른 프로젝트 간 스킬 충돌을 방지합니다.

채팅에서 슬래시 명령어를 직접 입력하여 스킬을 호출할 수 있습니다:
```
/skill-name
```

## 워크스페이스 패널 (파일 편집)

워크스페이스 패널은 채팅 오른쪽에 표시되는 코드 뷰어 및 파일 브라우저입니다. AI 에이전트 머신의 파일을 탐색·미리보기·편집할 수 있습니다(원격 서버 포함).

### 패널 열기

채팅 헤더의 **패널 전환 버튼**(분할 창 아이콘)을 클릭합니다. 오른쪽에 패널이 열리며 드래그 핸들로 크기를 조절할 수 있습니다.

### 파일 트리 (Files 탭)

**Files** 탭에는 룸 작업 디렉토리의 프로젝트 디렉토리 트리가 표시됩니다. 파일을 클릭하여 열 수 있습니다. 폴더는 클릭으로 펼치기/접기.

### 코드 뷰어

텍스트 파일은 **구문 강조**(highlight.js), **줄 번호**, 어두운 코드 배경으로 열립니다. 상단 경로 탐색(breadcrumb)에 파일 경로가 표시됩니다.

### 파일 편집

텍스트 파일의 **Edit** 버튼을 클릭하여 편집 모드로 진입합니다. **CodeMirror 6** 구문 강조 지원(JavaScript, TypeScript, Python, JSON, HTML, CSS, Markdown).

- **저장**: `Ctrl+S`(`Cmd+S`) 또는 Save 버튼
- **전체 화면**: 확장 버튼으로 에디터 전체 너비 사용(채팅 패널 숨김)
- **충돌 감지**: 편집 중 파일이 변경되면 다시 로드 또는 덮어쓰기 대화 상자 표시
- **자동 저장**: 저장하지 않은 편집은 브라우저에 저장되며, 다시 열 때 복구 프롬프트 표시
- **키보드 단축키**: `Ctrl+/`(주석 토글), `Ctrl+Shift+D`(줄 복제), `Alt+Shift+↑/↓`(줄 복사), `Tab`(들여쓰기)

CodeMirror 로드에 실패하면(CDN 접근 불가 등) 일반 텍스트 영역으로 폴백됩니다.

### 파일 작업 (컨텍스트 메뉴)

파일 트리에서 파일이나 폴더를 우클릭(태블릿에서는 길게 누르기)하여 컨텍스트 메뉴를 열 수 있습니다:

- **새 파일** — 선택한 폴더에 새 파일 생성
- **새 폴더** — 새 디렉토리 생성
- **이름 바꾸기** — 파일 또는 폴더 이름 변경
- **삭제** — 파일 삭제(확인 대화 상자 포함)

### Markdown 미리보기

`.md` 파일은 제목, 목록, 코드 블록, 표, 링크가 포함된 형식화된 마크다운으로 렌더링됩니다.

### 이미지 미리보기

이미지 파일(PNG, JPG, GIF, WebP, SVG)은 가운데 정렬된 미리보기로 표시됩니다. 원격 에이전트의 경우 에이전트의 WebSocket 연결을 통해 이미지를 가져옵니다.

### Git 차이 (Git 탭)

**Git** 탭에는 미커밋 변경 사항(`git diff`)이 초록/빨강 줄 강조, 파일 헤더, 변경 통계와 함께 표시됩니다.

### 클릭 가능한 파일 경로

AI 에이전트가 메시지에서 파일 경로(예: `/home/user/project/file.py`)를 언급하면 그 경로가 **클릭 가능**해집니다. 클릭하면 워크스페이스 패널에서 파일이 열립니다. 백틱 코드 블록과 코드 펜스 내의 경로에서도 동작합니다.

### 파일 다운로드

파일 트리의 파일에 마우스를 올리면 **다운로드 버튼**(화살표 아이콘)이 나타납니다. 클릭하면 로컬 기기에 다운로드됩니다. 원격 에이전트 파일도 WebSocket을 통해 가져와 브라우저 다운로드로 전달됩니다.

### 원격 파일 탐색

워크스페이스는 로컬과 원격 에이전트 모두에서 동작합니다. 원격 에이전트의 경우 파일 작업이 에이전트의 WebSocket 연결을 통해 프록시됩니다. 태블릿과 스마트폰을 포함한 모든 기기에서 에이전트가 실행 중인 모든 머신의 파일을 탐색할 수 있습니다.

## 자동화

Stoa는 **Slack 트리거 자동화**를 지원합니다 — Slack 이벤트가 조건과 일치할 때 자동으로 대상 룸에 프롬프트를 전송하는 규칙입니다.

### 연결 관리

**설정 > 자동화 > 연결**을 열어 Slack 연결을 관리합니다. 각 연결은 고유한 자격 증명을 가진 독립적인 Slack Socket Mode 세션입니다.

**+ add connection**을 클릭하여 새 연결을 추가합니다:

- **Name** — 연결을 식별하는 레이블 (예: "Qiscus 워크스페이스", "Support Bot")
- **Token Type** — `bot` (Bot Token `xoxb-...` 사용) 또는 `user` (User Token `xoxp-...` 사용)
- **App Token** (`xapp-1-...`) — Socket Mode WebSocket 연결용
- **Bot / User Token** — 이벤트를 수신하고 API 호출을 수행하는 토큰

각 연결 카드에는 현재 상태가 표시됩니다: **connecting**, **connected**, **disconnected**, 또는 **error**. 실패한 연결을 재시도하려면 **reconnect** 버튼을, 관리하려면 **edit**과 **delete** 버튼을 사용하세요.

Slack 앱 토큰 생성 방법은 [Slack 설정 가이드](doc-slack-setup)를 참조하세요.

### 자동화 규칙 만들기

하나 이상의 연결이 활성화되면 **+ new rule**을 클릭하여 규칙을 만듭니다:

- **이름** — 규칙의 설명 레이블
- **연결** — 이 규칙을 실행할 연결, 또는 모든 연결의 이벤트와 일치하는 **Any connection**
- **트리거 이벤트** — 어떤 Slack 이벤트가 규칙을 실행할지: `message`(공개 채널), `message.groups`(비공개 채널), `mention`, 또는 `reaction_added`
- **조건** — 선택적 필터: `message_text contains`, `message_text not_contains`, `message_text starts_with`, 또는 `matches_regex`. 여러 조건은 AND로 결합
- **대상 룸** — 프롬프트를 받을 Stoa 룸
- **프롬프트 템플릿** — 룸으로 보낼 메시지. 사용 가능한 변수: `{{slack_message_text}}`, `{{slack_message_link}}`, `{{slack_user}}`, `{{slack_channel}}`, `{{extracted_url}}`, `{{slack_thread_ts}}`

### 활성화 / 비활성화

각 규칙에는 활성화/비활성화 토글이 있습니다. 비활성화된 규칙은 Slack 이벤트가 조건과 일치해도 실행되지 않습니다.

## 설정

사이드바의 **톱니바퀴 아이콘**을 클릭하여 설정 패널을 엽니다. 설정은 여섯 개의 탭으로 구성됩니다:

### AI Agent

등록된 모든 에이전트, 온라인 상태, 버전, 작업 디렉터리, 스킬을 확인. 새 에이전트 추가, 이름 변경, 삭제, 재스캔, 강제 업데이트 가능.

### 서버

- **표시 이름** — 채팅에 표시되는 사용자 이름
- **아바타** — 프로필 이미지 업로드 (아바타 영역을 클릭하여 업로드 또는 삭제)
- **공개 URL** — 에이전트와 다른 디바이스가 서버에 접근하는 URL (Tailscale/원격 설정에서 중요)
- **포트** — 서버 포트 변경 (재시작 필요; [포트 변경 가이드](doc-port) 참조)
- **최대 AI 턴 수** — 사람 메시지당 에이전트 응답 최대 수 (무한 루프 방지)
- **동시 세션** — 에이전트가 병렬로 응답할 수 있는 룸 수 (즉시 적용, 재시작 불필요)
- **세션 유휴 TTL** — 유휴 AI 세션이 메모리를 해제하기 위해 자동으로 닫히기까지의 시간(분) (기본값: 5분)
- **정리 시각** — 일일 업로드 정리 실행 시각 (24시간 형식)
- **최대 파일 보관 기간** — 업로드 파일이 정리되기까지의 보관 시간 (시간 단위)

### 플랫폼

기본 제공 Claude 모델 외에 외부 AI 모델 제공자를 설정합니다. **+ add platform**을 클릭하여 새 제공자를 등록합니다:

- **Type** — **Ollama Cloud** (내장 프록시, URL 불필요) 또는 **Custom Platform** (OpenAI 호환 API)
- **Name** — 플랫폼 이름 (예: "My OpenRouter")
- **Base URL** — Custom Platform에 필요 (예: `https://openrouter.ai/api/v1`)
- **API Key** — 해당 제공자의 API 키

새 플랫폼의 경우 **add & discover**를 클릭하여 저장과 탐색을 한 번에 할 수 있습니다. 기존 플랫폼은 **discover models**를 클릭하여 재탐색합니다. Discovery는 모델별 기능(vision 👁 및 도구 호출 ⚙)을 감지합니다. 도구 호출 미지원 모델도 포함되어 사용 가능하지만 텍스트 전용 모드로 동작합니다.

**모델 활성화/비활성화**: Discovery 후 발견된 모든 모델의 체크리스트가 표시됩니다. 룸 선택기에 표시할 모델만 체크하고 **update**를 클릭하여 저장하세요. **select all / deselect all**로 일괄 전환도 가능합니다. 선택은 세션 간 유지되며 재탐색 시 자동 업데이트됩니다.

API 키는 서버에 저장되며 Settings 편집 폼에서만 브라우저로 반환됩니다 (싱글유저 앱이므로 접근 권한은 본인만 갖습니다).

**지원 제공자** — OpenAI 호환 API 엔드포인트라면 어디든 사용 가능합니다:
- **Ollama Cloud** — "Ollama Cloud" 타입 선택 (내장 프록시, 키 로테이션, 40+ 모델, 최대 480B)
- **Local Ollama** — `http://localhost:11434/v1` (무료, 프라이빗, 오프라인 가능 — [Ollama 설정 가이드](doc-ollama) 참조)
- **OpenRouter** — `https://openrouter.ai/api/v1`
- **Together AI** — `https://api.together.xyz/v1`
- **Groq** — `https://api.groq.com/openai/v1`

## 아키텍처 개요

```
브라우저  <-->  WebSocket  <-->  server.js  <-->  에이전트 (stoa.js)
                                    |                   |
                                 SQLite DB      Claude Code CLI
                                                  (+ 환경 변수를 통한
                                                   외부 플랫폼 연동)
```

- **server.js** — HTTP + WebSocket 서버. 룸, 메시지, 에이전트 오케스트레이션 관리
- **public/** — 프런트엔드 (빌드 단계 불필요)
- **stoa.js** — 각 에이전트 머신에서 실행되는 에이전트 클라이언트
- **claude-session.js** — 영구적인 Claude Code CLI 서브프로세스 관리
- **SQLite** — 모든 데이터를 `stoa.db`에 로컬 저장 (성능을 위한 WAL 모드)

모든 AI 모델은 **Claude Code CLI**를 통해 라우팅됩니다. 커스텀 외부 플랫폼(OpenRouter, Together AI 등)의 경우, 서버가 플랫폼별 환경 변수(`ANTHROPIC_BASE_URL`, `ANTHROPIC_AUTH_TOKEN`)를 CLI 프로세스에 전달합니다. **Ollama Cloud**의 경우, 요청은 Stoa 내장 프록시(`/v1/messages`)를 통해 라우팅되며 서버 측에서 API 키 로테이션을 처리합니다 — 에이전트는 Stoa에 연결하고 ollama.com에는 직접 연결하지 않습니다.

**프롬프트의 룸 컨텍스트 주입**: 에이전트가 트리거될 때마다 서버는 에이전트의 시스템 프롬프트에 `Room ID: <id>`를 주입합니다. 이를 통해 에이전트는 항상 자신이 어느 룸에서 동작하는지 알 수 있으며, ID를 명시적으로 전달하지 않고도 `sendProactiveMessage(roomId, ...)` 등의 룸 인식 작업을 수행할 수 있습니다.

## 팁

- **여러 에이전트, 하나의 룸**: 상호 보완적인 에이전트(예: 코더와 리뷰어)를 같은 룸에 배치하고 @멘션으로 협업시키세요
- **전용 룸**: 주제나 프로젝트별로 별도의 룸을 만드세요. 각 룸은 자체 대화 기록을 유지합니다
- **작업 디렉터리**: 다른 룸에 다른 작업 디렉터리를 할당하면 같은 에이전트가 여러 프로젝트에서 작업할 수 있습니다
- **파일 공유**: 파일을 채팅에 직접 드래그 앤 드롭하세요 — 에이전트가 즉시 읽을 수 있습니다
- **먼저 검색**: 에이전트에게 질문하기 전에 검색을 사용하여 다른 룸에서 이미 논의되었는지 확인하세요
