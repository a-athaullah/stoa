# ブラウザ設定ガイド

Stoa は特別な権限を必要とするブラウザ機能を使用します — 特にローカルネットワークや Tailscale 経由で HTTP（非 HTTPS）アクセスする場合。このガイドでは、デスクトップとモバイルで**音声入力**、**プッシュ通知**、**ホーム画面に追加（PWA）**を有効にする方法を説明します。

---

## 音声入力（音声テキスト変換）

Stoa のチャットコンポーザーにはマイクボタンが内蔵されています。クリックして話し始めると、音声がリアルタイムでテキストフィールドに書き起こされます。準備ができたら送信をクリックします。

### 要件

- **ブラウザ**：Chrome または Edge（完全サポート）。Safari は部分サポート。Firefox はサポートが限定的。
- **インターネット接続**：Chrome はバックグラウンドで Google のクラウド音声認識を使用します。
- **マイク権限**：ブラウザが Stoa のオリジンに対してマイクアクセスを許可する必要があります。

### HTTP でのマイク許可（デスクトップ）

Chrome はデフォルトで非 HTTPS オリジンのマイクアクセスをブロックします。LAN IP や Tailscale IP（例：`http://192.168.1.100:3001`）で Stoa にアクセスする場合は、ホワイトリストに追加する必要があります：

1. 新しいタブを開き、次のアドレスにアクセス：`chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. テキストフィールドに Stoa の URL を入力：`http://YOUR_IP:PORT`
3. ドロップダウンを **Enabled** に設定
4. **Relaunch** をクリックして Chrome を再起動

再起動後、Chrome はそのオリジンをセキュアとして扱い、マイクアクセスを許可します。

**ヒント**：サーバーが動作しているのと同じマシンから Stoa にアクセスする場合は、`localhost:PORT` を使用してください — Chrome は localhost を自動的にセキュアなオリジンとして扱うため、フラグは不要です。

### Android でのマイク許可（Chrome）

1. Chrome を開き、次のアドレスにアクセス：`chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. Stoa の URL を入力（例：`http://192.168.1.100:3001`）
3. **Enabled** に設定して **Relaunch** をタップ
4. Stoa に移動してマイクボタンをタップ — Chrome がマイク権限を求めるので、**許可**をタップ

### iOS でのマイク許可（Safari）

iOS Safari はセキュアなオリジンで音声認識をサポートします。HTTP アクセスの場合：

1. **設定 > Safari > 詳細 > 実験的な機能**を開く
2. **MediaDevices** と **SpeechRecognition** が有効になっていることを確認
3. Stoa がマイクアクセスを求めたら、**許可**をタップ

注意：iOS Safari の Web Speech API サポートは限定的な場合があります。最良の体験のためには、デスクトップブラウザまたは Android Chrome をお使いください。

### 音声コマンド

マイクがアクティブな間、ボタンをタップする代わりに音声コマンドを使用できます：

| 言語 | 送信 | 停止 | クリア |
|------|------|------|--------|
| EN | "send now" | "stop listening" | "clear all" |
| ID | "kirimkan sekarang" | "matikan mic" | "hapus semua" |
| JA | "送信して" | "マイク止め" | "全部消して" |
| KO | "지금 보내" | "마이크 끄" | "전부 지우" |
| ZH | "现在发送" | "关闭麦克风" | "全部清除" |

マイクボタンの横にある **EN/ID/JA/KO/ZH** ボタンで言語を切り替えられます（EN → ID → JA → KO → ZH）。選択は保存されます。

### 動作の違い：デスクトップ vs Android

音声入力はデスクトップと Android の両方で動作しますが、Chrome Android の制限により動作が異なります：

| 動作 | デスクトップ（Chrome/Edge） | Android（Chrome） |
|------|---------------------------|-------------------|
| **連続リスニング** | はい — マイクが無期限にオン | 制限あり — 短い無音（約3-7秒）後にマイクが停止 |
| **無音後** | マイクがシームレスに自動再起動 | マイクがオフになる — タップして再開 |
| **セッション間のテキスト** | 再起動をまたいで保持 | テキストフィールドに保持 — タップで続行 |
| **音声コマンド** | マイクオン中いつでも動作 | マイクアクティブ中（タイムアウト前）に動作 |
| **AI 応答中のマイク** | ミュート（テキスト書き込みなし） | 同様 — 処理中はミュート |

**なぜ違いがあるのか？** Android Chrome の Web Speech API は連続リスニングモードを適切にサポートしていません。Android でマイクを自動再起動すると、再起動のたびに「ディン」という音が鳴り、テキストが重複する場合があります。これらの問題を避けるため、Stoa は Android ではマイクが自然に停止するようにしています。

**Android でのヒント**：メッセージ全体を一気に話してから、一時停止する前に送信コマンド（例：JA では「送信して」）を言ってください。タイムアウトを回避し、一連の流れで送信できます。

### 言語サポート

音声入力のデフォルトは**英語（en-US）**です。英語、インドネシア語、日本語、韓国語、中国語の5言語に対応しています。マイクの横にある言語切り替えボタンで変更できます。

## プッシュ通知

Stoa は、タブがバックグラウンドにあってもエージェントが応答した際にブラウザのプッシュ通知を送信します。

### デスクトップでの有効化（Chrome/Edge）

1. **設定 > 一般 > 通知**を開き、通知を有効にする
2. ブラウザが「通知を許可しますか？」と表示したら、**許可**をクリック
3. 誤ってブロックした場合は、アドレスバーの**ロック/情報アイコン** > **サイトの設定** > 通知を**許可**に設定

### Android での有効化（Chrome）

1. Chrome で Stoa を開く
2. **設定 > 一般 > 通知**を開き、通知を有効にする
3. プロンプトが表示されたら、**許可**をタップ
4. ブロックした場合：アドレスバーの**ロックアイコン** > **権限** > **通知** > **許可**

別の方法：**Chrome 設定 > サイトの設定 > 通知** > Stoa の URL を見つけて**許可**に設定。

### iOS での有効化（Safari 16.4+）

iOS でのプッシュ通知には、まず Stoa を PWA としてインストールする必要があります：

1. **Stoa をホーム画面に追加**（次のセクションを参照）
2. ホーム画面のアイコンから Stoa を開く
3. **設定 > 一般 > 通知**を開き、通知を有効にする
4. プロンプトが表示されたら、**許可**をタップ

注意：iOS のプッシュ通知は PWA（ホーム画面アプリ）からのみ動作し、Safari のタブからは動作しません。

## ホーム画面に追加（PWA）

Stoa を Progressive Web App としてインストールすると、ネイティブアプリのような体験が得られます — フルスクリーン、ブラウザの装飾なし、素早いアクセス。

### Android（Chrome）

1. Chrome で Stoa を開く
2. **三点メニュー**（右上）をタップ
3. **「ホーム画面に追加」**または**「アプリをインストール」**をタップ
4. 名前を確認して**追加**をタップ
5. ホーム画面にアプリアイコンとして Stoa が表示されます

### iOS（Safari）

1. Safari で Stoa を開く
2. **共有ボタン**（四角から矢印が出たアイコン、下部中央）をタップ
3. スクロールして**「ホーム画面に追加」**をタップ
4. 名前を確認して**追加**をタップ
5. ホーム画面にアプリアイコンとして Stoa が表示されます

### デスクトップ（Chrome/Edge）

1. Chrome または Edge で Stoa を開く
2. アドレスバーの**インストールアイコン**をクリック（または三点メニュー > 「Stoa をインストール」）
3. **インストール**をクリック
4. Stoa がスタンドアロンウィンドウとして開きます

### PWA としてインストールする理由

- **フルスクリーン**：アドレスバーやブラウザタブがスペースを占有しない
- **素早いアクセス**：ネイティブアプリのようにホーム画面から起動
- **iOS でのプッシュ通知**：iOS のプッシュ通知サポートに必要
- **オフラインインジケーター**：接続状態を明確に表示
