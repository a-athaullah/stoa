# サーバーポートの変更

Stoa のデフォルトポートは **3030** です。変更するには、`config.yaml` の `port` を設定（または `config.yaml` を上書きする `.env` の `PORT`）してから、ゲートウェイを再起動します。

> Stoa はネイティブのバックグラウンドサービス —— **ゲートウェイ**（macOS は launchd、Linux は systemd）として動作します。PM2 は不要です。`stoa gateway <start|stop|restart|status>` で管理します。

> ⚠️ **接続済みのエージェントがある場合は、先にお読みください。**
>
> 各エージェントはインストール時にサーバー URL（ポート含む）を `STOA_URL` としてサービスユニットに保存しています。サーバーポートを変更しても、エージェントは**自動的に更新されません**。
>
> - 旧サーバーが停止すると、エージェントの WebSocket 接続は切断されます
> - 数秒ごとに再接続を試みますが、**旧ポート**に対してであり、再接続できません
> - 更新するまでオフラインのままになります
>
> ポート変更後、**各エージェントのマシンを更新してください** —— ステップ3を参照。

---

## ステップ1：ポートを設定

データディレクトリの `config.yaml`（インストール時は `~/.stoa/server/config.yaml`、開発時はリポジトリのルート）を編集します：

```yaml
port: 3031
```

（または `.env` に `PORT=3031` を設定 —— 環境変数が `config.yaml` を上書きします。）

---

## ステップ2：ゲートウェイを再起動

```bash
stoa gateway restart
```

その後、サーバーは `http://localhost:3031` でアクセス可能になります。

---

## ステップ3：各エージェントマシンを更新

各エージェントはサービスユニットに `STOA_URL` が埋め込まれています。最も簡単なのは、そのエージェントの**インストールコマンドを再実行**（設定 → エージェントに表示）して、新しい URL で再登録することです。

その場で更新する場合は、エージェントのサービスユニットを編集して再起動します：

- macOS: `~/Library/LaunchAgents/com.stoa.agent.<id>.plist`
- Linux: `~/.config/systemd/user/stoa-agent-<id>.service`

`STOA_URL` の値を新しいポートに変更し、サービスをリロードします。

---

## ステップ4：パブリック URL の更新（設定済みの場合）

パブリック URL を設定している場合は、`config.yaml`（`public_url`）または Web UI の**設定**で更新してください —— 例：`http://100.x.x.x:3030` → `http://100.x.x.x:3031`。
