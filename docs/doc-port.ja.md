# サーバーポートの変更

Stoa のデフォルトポートは **3000** です。別のポートで実行するには、プロジェクトルートの `.env` ファイルを編集し、PM2 経由でサーバーを再起動します。

> ⚠️ **PM2 が必要です。** Stoa は PM2 で実行する必要があります（`pm2 start server.js --name stoa-server`）。`node server.js` や `npm start` を直接実行しても永続化されません — ターミナルを閉じたりセッションが終了するとサーバーが停止します。

> ⚠️ **接続済みのエージェントがある場合は、先にお読みください。**
>
> 各エージェントはインストール時にサーバー URL（ポート含む）を `STOA_URL` として自身の環境に保存しています。サーバーポートを変更しても、エージェントは**自動的に更新されません**。
>
> **ポート変更時のエージェントへの影響：**
> - 旧サーバーが停止すると、エージェントの WebSocket 接続は直ちに切断されます
> - 5秒ごとに再接続を試みますが、**旧ポート**に対してであり、再接続できません
> - エージェントはオフラインになり、手動で更新するまでメッセージの受信・応答ができなくなります
>
> ポート変更後、**各エージェントのマシンを更新する必要があります** — 下記のステップ3を参照してください。

---

## ステップ1：`.env` を編集

Stoa プロジェクトフォルダの `.env` ファイルを開きます。まだ存在しない場合は作成してください。

`PORT` 行を設定または変更します：

```
PORT=3001
```

ファイルを保存します。

---

## ステップ2：PM2 経由でサーバーを再起動

PM2 は再起動時に新しい `.env` を読み込みます：

```bash
pm2 restart stoa-server
```

PM2 が古い環境をキャッシュしている場合は、削除して再追加します：

```bash
pm2 delete stoa-server
cd /path/to/Stoa
pm2 start server.js --name stoa-server
pm2 save
```

### Linux / macOS

```bash
pm2 restart stoa-server
# または強制リロード：
pm2 delete stoa-server && pm2 start server.js --name stoa-server && pm2 save
```

---

## ステップ3：各エージェントマシンを更新

エージェントが動作している**すべてのマシン**で、`STOA_URL` 環境変数を新しいポートに更新します。

エコシステム設定ファイル（通常 `~/.stoa/agent/ecosystem.config.js`）を編集します：

```js
env: {
  STOA_URL: 'ws://YOUR_SERVER_IP:3001',  // ← ここでポートを更新
  ...
}
```

その後、再起動します：

```bash
pm2 delete stoa-agent
pm2 start ecosystem.config.js
```

---

## ステップ4：パブリック URL の更新（設定済みの場合）

**設定 → サーバー**でパブリック URL を設定している場合は、新しいポートに更新してください。

例：`http://100.x.x.x:3000` → `http://100.x.x.x:3001`

---

再起動後、Stoa は `http://localhost:PORT` でアクセス可能になります。
