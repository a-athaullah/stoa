# 浏览器设置指南

Stoa 使用需要特殊权限的浏览器功能 — 尤其是通过本地网络或 Tailscale 以 HTTP（非 HTTPS）方式访问时。本指南介绍如何在桌面端和移动端启用**语音输入**、**推送通知**和**添加到主屏幕（PWA）**。

---

## 语音输入（语音转文字）

Stoa 的聊天编辑器内置了麦克风按钮。点击后开始说话 — 语音会实时转写到文本框中。准备好后点击发送。

### 要求

- **浏览器**：Chrome 或 Edge（完全支持）。Safari 部分支持。Firefox 支持有限。
- **互联网连接**：Chrome 在后台使用 Google 的云端语音识别。
- **麦克风权限**：浏览器必须允许 Stoa 来源的麦克风访问。

### 在 HTTP 上允许麦克风（桌面端）

Chrome 默认阻止非 HTTPS 来源的麦克风访问。如果您通过局域网 IP 或 Tailscale IP（例如 `http://100.90.197.4:3001`）访问 Stoa，需要将其加入白名单：

1. 打开新标签页，访问：`chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. 在文本框中输入您的 Stoa URL：`http://YOUR_IP:PORT`
3. 将下拉菜单设为 **Enabled**
4. 点击 **Relaunch** 重启 Chrome

重启后，Chrome 会将该来源视为安全来源，允许麦克风访问。

**提示**：如果从服务器所在的同一台机器访问 Stoa，使用 `localhost:PORT` 即可 — Chrome 自动将 localhost 视为安全来源，无需设置标志。

### 在 Android 上允许麦克风（Chrome）

1. 打开 Chrome，访问：`chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. 输入 Stoa URL（例如 `http://100.90.197.4:3001`）
3. 设为 **Enabled** 并点击 **Relaunch**
4. 导航到 Stoa，点击麦克风按钮 — Chrome 会请求麦克风权限，点击**允许**

### 在 iOS 上允许麦克风（Safari）

iOS Safari 在安全来源上支持语音识别。对于 HTTP 访问：

1. 打开**设置 > Safari > 高级 > 实验性功能**
2. 确保 **MediaDevices** 和 **SpeechRecognition** 已启用
3. 当 Stoa 请求麦克风访问时，点击**允许**

注意：iOS Safari 的 Web Speech API 支持可能有限。为获得最佳体验，请使用桌面浏览器或 Android Chrome。

### 语音命令

麦克风激活时，您可以使用语音命令代替点击按钮：

| 语言 | 发送 | 停止 | 清除 |
|------|------|------|------|
| EN | "send now" | "stop listening" | "clear all" |
| ID | "kirimkan sekarang" | "matikan mic" | "hapus semua" |
| JA | "送信して" | "マイク止め" | "全部消して" |
| KO | "지금 보내" | "마이크 끄" | "전부 지우" |
| ZH | "现在发送" | "关闭麦克风" | "全部清除" |

点击麦克风按钮旁的 **EN/ID/JA/KO/ZH** 按钮循环切换语言（EN → ID → JA → KO → ZH）。您的选择会被保存。

### 行为差异：桌面端 vs Android

语音输入在桌面端和 Android 上都可使用，但由于 Chrome Android 的限制，行为有所不同：

| 行为 | 桌面端（Chrome/Edge） | Android（Chrome） |
|------|---------------------|-------------------|
| **持续监听** | 是 — 麦克风无限期保持开启 | 有限 — 短暂静默（约3-7秒）后麦克风停止 |
| **静默后** | 麦克风无缝自动重启 | 麦克风关闭 — 点击麦克风恢复 |
| **会话间文本** | 跨重启保留 | 在文本框中保留 — 点击麦克风继续 |
| **语音命令** | 麦克风开启时随时可用 | 麦克风活跃时（超时前）可用 |
| **AI 回复期间的麦克风** | 静音（不写入文本） | 相同 — 处理期间静音 |

**为什么有差异？** Android Chrome 的 Web Speech API 不能正确支持持续监听模式。在 Android 上自动重启麦克风会导致每次重启时发出"叮"的声音，并可能产生重复文本。为避免这些问题，Stoa 让麦克风在 Android 上自然停止。

**Android 使用技巧**：一口气说完整条消息，然后在停顿前说发送命令（例如中文模式下说"现在发送"）。这样可以避免超时，一次性完成发送。

### 语言支持

语音输入默认为**英语（en-US）**。支持五种语言：英语、印尼语、日语、韩语和中文。使用麦克风旁的语言切换按钮进行切换。

## 推送通知

即使标签页在后台，Stoa 也会在代理回复时发送浏览器推送通知。

### 桌面端启用（Chrome/Edge）

1. 点击 Stoa 侧边栏底部的**铃铛图标**
2. 当浏览器提示"是否允许通知？"时，点击**允许**
3. 如果不小心阻止了，点击地址栏的**锁定/信息图标** > **网站设置** > 将通知设为**允许**

### Android 启用（Chrome）

1. 在 Chrome 中打开 Stoa
2. 点击侧边栏底部的**铃铛图标**
3. 出现提示时，点击**允许**
4. 如果被阻止：点击地址栏的**锁定图标** > **权限** > **通知** > **允许**

或者：**Chrome 设置 > 网站设置 > 通知** > 找到 Stoa URL 并设为**允许**。

### iOS 启用（Safari 16.4+）

iOS 上的推送通知需要先将 Stoa 安装为 PWA：

1. **将 Stoa 添加到主屏幕**（见下一节）
2. 从主屏幕图标打开 Stoa
3. 点击铃铛图标启用通知
4. 出现提示时，点击**允许**

注意：iOS 推送通知仅从 PWA（主屏幕应用）中有效，不适用于 Safari 标签页。

## 添加到主屏幕（PWA）

将 Stoa 安装为 Progressive Web App 可获得原生应用般的体验 — 全屏显示、无浏览器框架、更快访问。

### Android（Chrome）

1. 在 Chrome 中打开 Stoa
2. 点击**三点菜单**（右上角）
3. 点击**"添加到主屏幕"**或**"安装应用"**
4. 确认名称并点击**添加**
5. Stoa 以应用图标形式出现在主屏幕上

### iOS（Safari）

1. 在 Safari 中打开 Stoa
2. 点击**分享按钮**（带箭头的方框，底部居中）
3. 向下滚动并点击**"添加到主屏幕"**
4. 确认名称并点击**添加**
5. Stoa 以应用图标形式出现在主屏幕上

### 桌面端（Chrome/Edge）

1. 在 Chrome 或 Edge 中打开 Stoa
2. 点击地址栏中的**安装图标**（或三点菜单 > "安装 Stoa"）
3. 点击**安装**
4. Stoa 以独立窗口打开

### 为什么要安装为 PWA？

- **全屏**：没有地址栏或浏览器标签占用空间
- **快速访问**：像原生应用一样从主屏幕启动
- **iOS 推送通知**：iOS 推送通知支持的必要条件
- **离线指示器**：清晰显示连接状态
