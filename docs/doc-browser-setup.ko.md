# 브라우저 설정 가이드

Stoa는 특별한 권한이 필요한 브라우저 기능을 사용합니다 — 특히 로컬 네트워크나 Tailscale을 통해 HTTP(비HTTPS)로 접근할 때. 이 가이드에서는 데스크톱과 모바일에서 **음성 입력**, **푸시 알림**, **홈 화면에 추가(PWA)**를 활성화하는 방법을 다룹니다.

---

## 음성 입력 (음성-텍스트 변환)

Stoa의 채팅 컴포저에는 마이크 버튼이 내장되어 있습니다. 클릭하고 말하면 음성이 실시간으로 텍스트 필드에 변환됩니다. 준비되면 전송을 클릭합니다.

### 요구 사항

- **브라우저**: Chrome 또는 Edge (완전 지원). Safari는 부분 지원. Firefox는 제한적 지원.
- **인터넷 연결**: Chrome은 백그라운드에서 Google의 클라우드 음성 인식을 사용합니다.
- **마이크 권한**: 브라우저가 Stoa의 오리진에 대해 마이크 접근을 허용해야 합니다.

### HTTP에서 마이크 허용 (데스크톱)

Chrome은 기본적으로 비HTTPS 오리진에서 마이크 접근을 차단합니다. LAN IP나 Tailscale IP(예: `http://192.168.1.100:3001`)로 Stoa에 접근하는 경우 화이트리스트에 추가해야 합니다:

1. 새 탭을 열고 다음으로 이동: `chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. 텍스트 필드에 Stoa URL 입력: `http://YOUR_IP:PORT`
3. 드롭다운을 **Enabled**로 설정
4. **Relaunch**를 클릭하여 Chrome 재시작

재시작 후 Chrome은 해당 오리진을 안전한 것으로 취급하고 마이크 접근을 허용합니다.

**팁**: 서버가 실행 중인 동일한 머신에서 Stoa에 접근하는 경우 `localhost:PORT`를 사용하세요 — Chrome은 localhost를 자동으로 안전한 오리진으로 취급하므로 플래그가 필요 없습니다.

### Android에서 마이크 허용 (Chrome)

1. Chrome을 열고 다음으로 이동: `chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. Stoa URL 입력 (예: `http://192.168.1.100:3001`)
3. **Enabled**로 설정하고 **Relaunch** 탭
4. Stoa로 이동하여 마이크 버튼 탭 — Chrome이 마이크 권한을 요청하면 **허용** 탭

### iOS에서 마이크 허용 (Safari)

iOS Safari는 안전한 오리진에서 음성 인식을 지원합니다. HTTP 접근의 경우:

1. **설정 > Safari > 고급 > 실험적 기능** 열기
2. **MediaDevices**와 **SpeechRecognition**이 활성화되어 있는지 확인
3. Stoa가 마이크 접근을 요청하면 **허용** 탭

참고: iOS Safari의 Web Speech API 지원은 제한적일 수 있습니다. 최상의 경험을 위해 데스크톱 브라우저나 Android Chrome을 사용하세요.

### 음성 명령

마이크가 활성화된 동안 버튼을 탭하는 대신 음성 명령을 사용할 수 있습니다:

| 언어 | 전송 | 중지 | 지우기 |
|------|------|------|--------|
| EN | "send now" | "stop listening" | "clear all" |
| ID | "kirimkan sekarang" | "matikan mic" | "hapus semua" |
| JA | "送信して" | "マイク止め" | "全部消して" |
| KO | "지금 보내" | "마이크 끄" | "전부 지우" |
| ZH | "现在发送" | "关闭麦克风" | "全部清除" |

마이크 버튼 옆의 **EN/ID/JA/KO/ZH** 버튼으로 언어를 순환합니다 (EN → ID → JA → KO → ZH). 선택은 저장됩니다.

### 동작 차이: 데스크톱 vs Android

음성 입력은 데스크톱과 Android 모두에서 작동하지만 Chrome Android의 제한으로 동작이 다릅니다:

| 동작 | 데스크톱 (Chrome/Edge) | Android (Chrome) |
|------|----------------------|-------------------|
| **연속 듣기** | 예 — 마이크가 무기한 켜져 있음 | 제한적 — 잠시 침묵(~3-7초) 후 마이크 중지 |
| **침묵 후** | 마이크 자동 재시작 매끄럽게 | 마이크 꺼짐 — 탭하여 재개 |
| **세션 간 텍스트** | 재시작 간 보존 | 텍스트 필드에 보존 — 탭하여 계속 |
| **음성 명령** | 마이크 켜져 있는 동안 언제든 작동 | 마이크 활성 중(타임아웃 전) 작동 |
| **AI 응답 중 마이크** | 음소거 (텍스트 쓰기 없음) | 동일 — 처리 중 음소거 |

**왜 차이가 있나요?** Android Chrome의 Web Speech API는 연속 듣기 모드를 제대로 지원하지 않습니다. Android에서 마이크를 자동 재시작하면 매번 "딩" 소리가 나고 텍스트가 중복될 수 있습니다. 이러한 문제를 방지하기 위해 Stoa는 Android에서 마이크가 자연스럽게 중지되도록 합니다.

**Android 팁**: 전체 메시지를 한 번에 말한 다음, 멈추기 전에 전송 명령(예: KO에서는 "지금 보내")을 말하세요. 타임아웃을 피하고 한 번에 전송할 수 있습니다.

### 언어 지원

음성 입력 기본값은 **영어 (en-US)**입니다. 영어, 인도네시아어, 일본어, 한국어, 중국어 등 5개 언어를 지원합니다. 마이크 옆의 언어 전환 버튼으로 변경할 수 있습니다.

## 푸시 알림

Stoa는 탭이 백그라운드에 있어도 에이전트가 응답할 때 브라우저 푸시 알림을 보냅니다.

### 데스크톱에서 활성화 (Chrome/Edge)

1. Stoa 사이드바 푸터의 **종 아이콘** 클릭
2. 브라우저가 "알림을 허용하시겠습니까?"라고 표시되면 **허용** 클릭
3. 실수로 차단한 경우 주소 표시줄의 **잠금/정보 아이콘** > **사이트 설정** > 알림을 **허용**으로 설정

### Android에서 활성화 (Chrome)

1. Chrome에서 Stoa 열기
2. 사이드바 푸터의 **종 아이콘** 탭
3. 프롬프트가 나타나면 **허용** 탭
4. 차단된 경우: 주소 표시줄의 **잠금 아이콘** > **권한** > **알림** > **허용**

또는: **Chrome 설정 > 사이트 설정 > 알림** > Stoa URL을 찾아 **허용**으로 설정.

### iOS에서 활성화 (Safari 16.4+)

iOS에서 푸시 알림을 받으려면 먼저 Stoa를 PWA로 설치해야 합니다:

1. **Stoa를 홈 화면에 추가** (다음 섹션 참조)
2. 홈 화면 아이콘에서 Stoa 열기
3. 종 아이콘을 탭하여 알림 활성화
4. 프롬프트가 나타나면 **허용** 탭

참고: iOS 푸시 알림은 PWA(홈 화면 앱)에서만 작동하며, Safari 탭에서는 작동하지 않습니다.

## 홈 화면에 추가 (PWA)

Stoa를 Progressive Web App으로 설치하면 네이티브 앱과 같은 경험을 얻을 수 있습니다 — 전체 화면, 브라우저 크롬 없음, 빠른 접근.

### Android (Chrome)

1. Chrome에서 Stoa 열기
2. **점 세 개 메뉴** (오른쪽 상단) 탭
3. **"홈 화면에 추가"** 또는 **"앱 설치"** 탭
4. 이름 확인 후 **추가** 탭
5. 홈 화면에 앱 아이콘으로 Stoa가 나타납니다

### iOS (Safari)

1. Safari에서 Stoa 열기
2. **공유 버튼** (사각형에서 화살표, 하단 중앙) 탭
3. 스크롤하여 **"홈 화면에 추가"** 탭
4. 이름 확인 후 **추가** 탭
5. 홈 화면에 앱 아이콘으로 Stoa가 나타납니다

### 데스크톱 (Chrome/Edge)

1. Chrome 또는 Edge에서 Stoa 열기
2. 주소 표시줄의 **설치 아이콘** 클릭 (또는 점 세 개 메뉴 > "Stoa 설치")
3. **설치** 클릭
4. Stoa가 독립 창으로 열립니다

### PWA로 설치하는 이유

- **전체 화면**: 주소 표시줄이나 브라우저 탭이 공간을 차지하지 않음
- **빠른 접근**: 네이티브 앱처럼 홈 화면에서 실행
- **iOS 푸시 알림**: iOS 푸시 알림 지원에 필요
- **오프라인 표시기**: 연결 상태를 명확히 표시
