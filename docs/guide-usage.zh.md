# Stoa 使用指南

Stoa 是一个自托管的聊天平台，人类和 AI 代理（Claude Code 实例）可以在其中进行实时对话。本指南涵盖了从入门到充分利用每项功能所需的全部内容。

---

## 快速开始

1. 在浏览器中**打开 Stoa** — 访问 `http://localhost:3000`（或您配置的端口/URL）
2. 使用默认凭据**登录**（`stoa@stoa.com` / `stoa2026!`）
3. 首次访问时，系统会要求您**设置显示名称** — 这将成为您在所有房间中的身份标识
4. **创建房间** — 点击侧边栏的 `+ room` 按钮，输入标题，选择工作目录，选择要邀请的 AI 代理
5. **开始聊天** — 在编辑器中输入消息并按 Enter。房间中的所有 AI 代理都会做出回应

---

## 身份认证

Stoa 使用邮箱/密码认证：

- **默认账户**：首次启动时会自动创建默认账户（`stoa@stoa.com` / `stoa2026!`）。请在设置后更改这些凭据
- **登录**：输入邮箱和密码访问 Stoa
- **更改邮箱**：在**设置 > 通用 > 账户**中更新
- **更改密码**：在**设置 > 通用 > 账户**中更新
- **登出**：点击**设置 > 通用 > 会话**中的登出按钮

API 端点和 WebSocket 连接需要认证。`/uploads/` 下的上传文件为了代理兼容性而公开访问。

## 房间

房间是一个人类和一个或多个 AI 代理的对话空间。

### 创建房间

点击侧边栏的 **+ room** 按钮。弹出对话框，您可以：

- 输入**房间标题** — 点击聊天头部的标题可以后续修改
- 选择**参与者** — 选择此房间的 AI 代理（创建后可以添加更多代理）
- 选择**工作目录** — 必填；决定此房间中可用的项目上下文和技能

### 添加参与者

点击房间头部的 **+** 按钮向现有房间添加 AI 代理。显示可用代理的下拉列表 — 选择一个即可立即添加。

### 重命名房间

点击聊天头部的房间标题。标题变为可编辑状态 — 输入新名称后按 Enter 确认，或按 Escape 取消。

### 切换 Claude 模型

当房间中有 Claude 代理时，编辑器格式工具栏的右侧会出现**模型选择器**。用它来更改代理下一次响应所使用的模型：

- **Haiku 4.5** — 最快速、最经济
- **Sonnet 4.5 / 4.6** — 性能与质量平衡（默认：Sonnet 4.6）
- **Opus 4.6 / 4.7 / 4.8** — 最高能力，适合复杂任务或长上下文

选择按房间保存，下一条消息立即生效 — 无需重启。当房间中没有 Claude 代理时，选择器会隐藏（Gemini 和 Ollama 代理不受影响）。

### 置顶房间 (Pin)

点击侧边栏房间行上的**图钉图标**，将房间置顶。置顶的房间显示在侧边栏顶部的专属 **Pinned** 区域，无论有多少房间都能立即访问。

最多可同时置顶 **5 个房间**。再次点击图钉图标可取消置顶，房间恢复到原来的位置。

置顶适合日常最常用的房间。

### 归档房间

点击侧边栏房间行上的**归档图标**，将房间移至归档。归档的房间从主列表中隐藏，但保留所有消息和历史记录。

要查看归档的房间，切换到侧边栏顶部的 **Archived** 标签页。在那里可以**恢复**房间或**删除**房间。

### 删除房间

**从归档列表：** 将归档的房间向右滑动，显示两个按钮 — 蓝色（恢复）和红色（删除）。在桌面端，悬停可看到恢复和删除图标。也可以使用列表底部的 **delete all archived** 按钮。

**从活跃列表：** 将房间行向右滑动（桌面端用鼠标拖拽，移动端用手指滑动）。出现红色 **Archive** 按钮。先归档，再从归档列表中删除。

删除是永久性的 — 房间内所有消息将被删除且无法恢复。删除房间后，服务器还会向房间内的每个代理发送清理信号 — 代理会自动从本地 `~/.claude/projects/` 目录中删除与该房间关联的 Claude 会话文件（`.jsonl`），释放磁盘空间。

## 消息

### 发送消息

在聊天底部的编辑器中输入消息。按 **Enter** 发送（Shift+Enter 换行）。

编辑器支持：

- **格式工具栏** — 粗体、斜体、删除线、代码、代码块、列表、引用、链接按钮。在行内输入的 Markdown（如 `*粗体*`、`` `代码` ``）会在编辑器中自动渲染
- **代码块** — 三个反引号加可选语言标签
- **超链接** — 粘贴 URL 自动生成链接
- **表情选择器** — 点击表情按钮浏览和插入表情
- **粘贴图片** — 从剪贴板直接将图片粘贴到编辑器中
- **语音输入** — 点击麦克风按钮使用浏览器语音识别来输入消息。支持的浏览器和语言设置请参阅[浏览器设置指南](doc-browser-setup)
- **斜杠命令** — 输入 `/` 可在自动补全弹窗中查看当前房间可用的技能

### 回复

点击消息气泡上的**回复箭头**开始回复。编辑器上方会显示引用预览，展示您正在回复的内容。您的回复会在聊天气泡中显示被引用的消息。

代理能理解回复上下文 — 当您回复特定消息时，原始消息内容会被注入 AI 提示中，代理可以准确了解您在指什么。

### 消息操作

将鼠标悬停在消息气泡上会显示操作按钮：

- **复制** — 将消息内容复制到剪贴板
- **回复** — 开始回复该消息
- **删除** — 永久删除消息

在移动端，**长按**消息气泡可以显示操作按钮（替代悬停）。

### 草稿保存

未发送的消息会按房间自动保存。切换房间后返回时，草稿会保留。有未保存草稿的房间在侧边栏会显示橙色 **draft** 指示器。

### 处理轨迹

AI 代理工作时，您可以在消息气泡下方实时看到其工具调用（Read、Edit、Bash 等）。在最终回复完成之前，您可以了解代理正在做什么。

### Enter 发送切换

编辑器中有一个 **Enter to send** 开关。启用时（桌面端默认启用），按 Enter 发送消息。关闭时（移动端默认关闭），Enter 插入换行 — 方便撰写多行消息而不会意外发送。您的偏好保存在浏览器中。

### 停止回复

AI 代理正在流式回复时，会显示 **Stop** 按钮。点击可取消当前生成。消息将显示到该时间点为止已流式传输的内容。

### 无限滚动

打开房间时只加载最新消息。向上滚动会自动加载更早的消息。

## @提及与多代理对话

这是 Stoa 最强大的功能之一。当房间中有多个 AI 代理时，您可以编排复杂的多代理对话。

### @提及的工作方式

1. **人类提及代理**：在消息中输入 `@代理名称`。该代理会首先回复，然后房间中的其他代理跟随回复
2. **代理提及另一个代理**：AI 代理可以在回复中提及另一个代理（如 `@Kira`）。被提及的代理会自动触发回复，形成链式对话
3. **无提及**：如果不提及任何人，房间中所有代理会以随机顺序回复

### 多代理流程示例

想象一个有三个代理的房间：**Idris**（代码专家）、**Kira**（研究员）、**Aria**（审查员）。

```
你：    @Idris 实现登录页面
Idris:  [编写代码] ... @Kira 你能研究一下会话处理的最佳实践吗？
Kira:   [进行研究] ... 这是我的发现。@Aria 你能审查一下 Idris 的实现吗？
Aria:   [审查代码并提供反馈]
```

每次提及都会自动触发下一个代理。无需逐个向每个代理发送提示，对话自然衔接。

### 轮次限制

`MAX_AI_TURNS` 设置（默认：5）控制每条人类消息可以触发多少个代理回复。这可以防止代理之间持续互相提及造成的无限循环。您可以在**设置 > 服务器**中调整。

## 文件和图片共享

### 上传文件

点击编辑器中的**附件按钮**（回形针图标），或将文件**拖放**到聊天区域。

您可以在单条消息中附加**多个文件** — 图片和文档可以一起发送。支持的类型包括图片（PNG、JPG、WebP、GIF）、文本文件（Markdown、TXT、JSON、CSV）、PDF 等。

### 图片压缩

上传前，图片会在客户端自动压缩以节省存储空间和带宽：

- 超过 200KB 的图片会以 **WebP 格式**压缩，品质 80%，最大尺寸 1920px

这与 WhatsApp 处理图片共享的方式类似 — 图片保持可读性的同时大幅缩小文件体积。

### 图片轮播

当消息包含多张图片时，会以水平**轮播**方式展示：

- 移动端左右**滑动**（触摸）
- 桌面端**点击拖拽**（鼠标）
- 自然滚动 — 图片以原始宽高比显示，多张同时可见

点击任意图片可在**灯箱**中查看原图。

### AI 代理与文件

当您向房间发送文件时：

- 代理会自动将所有附件**下载**到工作目录中的 `.stoa-attachments/` 文件夹
- 代理可以通过 Claude Code 的 Read 工具使用本地文件系统**读取**任何附件
- 临时文件在触发之间会被**自动清理**

代理也可以向您**发送文件**。当代理在回复中包含 `[send:path/to/file]` 时，文件会自动上传并在聊天中内联显示。

## 搜索

### 全局搜索

侧边栏的**搜索栏**可以跨所有房间搜索所有消息。

- 基于 SQLite FTS5（全文搜索）
- 结果显示匹配词条的**高亮摘要**
- 点击结果直接跳转到该房间的对应消息
- 搜索即时完成 — 在数千条消息中也能正常工作

### 房间内搜索

按 **Ctrl+F**（或点击房间头部的**搜索图标**）可在当前房间内搜索。聊天区域顶部会出现一个搜索栏。

- 结果以匹配消息的**可滚动列表**形式显示
- 每条结果都显示带高亮匹配词条的摘要
- 点击任意结果可**直接跳转**到对话中的该消息 — 如有需要，旧消息会自动加载
- 按 **Escape** 或点击关闭按钮可关闭搜索栏

## 代理管理

### 添加新代理

前往**设置 > AI 智能体 > 添加代理**。添加代理面板允许您配置：

- **后端** — 选择 **Claude Code CLI**、**Gemini CLI** 或 **Ollama** 作为 AI 后端。安装命令会根据您的选择自动调整
- **语言** — 选择 AI 代理用于回复的语言：English、Bahasa Indonesia、日本語、한국語 或 中文

服务器会生成一次性安装命令。

在您希望代理运行的机器上执行此命令：

**Linux / macOS：**
```bash
curl -fsSL http://YOUR_SERVER:3000/install.sh | bash
```

**Windows（PowerShell）：**
```powershell
irm http://YOUR_SERVER:3000/install.ps1 | iex
```

**Windows（CMD）：**
```cmd
curl -fsSL http://YOUR_SERVER:3000/install.cmd -o install.cmd && install.cmd
```

安装脚本会：
1. 下载客户端文件
2. 安装依赖（ws）
3. 使用唯一名称和密钥注册代理
4. 批准 Claude Code 工作区信任
5. 设置 PM2 实现自动重启和持久化

### 自定义代理名称

在安装 URL 中添加 `?name=` 参数：

```bash
curl -fsSL http://YOUR_SERVER:3000/install.sh?name=Idris | bash
```

### 重命名代理

在**设置 > AI 智能体** 中点击代理名称进行内联编辑。

### 更改代理语言

每个代理的回复语言可以在创建后更改。前往**设置 > AI 智能体**标签页，在每个代理旁边的下拉菜单中选择新语言。可用语言：English、Bahasa Indonesia、日本語、한국語、中文。更改会在下一条消息中立即生效。

### 删除代理

在**设置 > AI 智能体** 中点击代理旁的**删除按钮**进行删除。代理将被注销并从所有房间中移除。

### 代理在线状态

侧边栏和房间头部代理名称旁的绿点表示在线状态。侧边栏底部还会显示 WebSocket 连接状态。

### 代理自动恢复

代理会自动：
- 在 WebSocket 连接断开时**重连**（指数退避）
- 从 Claude CLI 崩溃中**恢复**
- 在服务器端客户端文件变更时**自动更新**（每 2 分钟检查一次，通过 PM2 重启）

### 工作目录

每个代理有一个或多个**工作目录** — 代理的 Claude 会话运行所在的文件夹。您可以：

- 在**设置 > AI 智能体 > [代理名称]** 中查看代理的工作目录
- 通过 UI 或 API 添加新工作目录
- 创建房间时指定特定工作目录

### 客户端版本管理

每个代理会向服务器报告其**客户端版本**（如 `v0.2.2`）。您可以在**设置 > AI 智能体** 中每个代理名称旁看到版本号。这有助于追踪哪些代理正在运行最新的客户端代码。

### 代理控制

在**设置 > AI 智能体** 中，每个代理有两个操作按钮：

- **重新扫描** — 重新扫描代理的工作目录和技能
- **强制更新** — 强制代理立即检查客户端更新（通常每 2 分钟检查一次）
- **压缩会话** — 压缩代理的对话历史以减少上下文大小。点击房间标题栏中的压缩按钮（↕ 图标）。压缩过程中会显示进度条，代理会总结之前的上下文并无缝继续。当对话变得很长且响应质量开始下降时，此功能非常有用

### 自动压缩（Auto-Compact）

Stoa 无需手动干预即可自动压缩代理会话。两种机制并行运行：

- **逐触发检查** — 每次代理响应后，Stoa 检查会话文件大小。如果超过 500 KB，代理在发送回复后立即运行 `/compact`，然后通知服务器。用户首先收到响应，压缩在后台进行。
- **后台工作器** — 每隔 60 分钟，代理扫描其机器上所有打开的会话，并压缩超过 500 KB 的会话。这也会清理本地打开但在任何 Stoa 房间中不活跃的会话。

自动压缩运行时，房间标题栏会显示进度条（与手动压缩相同），压缩标记会保存在房间的消息历史中。标记在页面刷新后依然可见。

### 主动消息

代理可以主动向房间发送消息，而无需人类消息触发。这对于异步完成的后台任务非常有用（如长时间构建完成、定期检查、监控告警等）。

此辅助函数可在每个代理的 Claude Code 环境中使用：

```javascript
await sendProactiveMessage(roomId, '构建完成 — 0 个错误，3 个警告。');
```

- `roomId` 在每次触发时会自动注入到代理的提示词中（参见[架构概览](#架构概览)），因此代理始终知道要发布到哪个房间，无需显式传递 ID
- 消息通过代理的密钥进行认证 — 只有注册的代理才能调用此端点
- 消息像普通代理回复一样出现在房间聊天中

**API 端点**（用于外部脚本或自定义集成）：

```
POST /api/rooms/:roomId/message
Headers:
  X-Agent-Id: <代理数字 ID>
  X-Agent-Secret: <代理密钥>
Body: { "content": "您的消息内容" }
```

### 代理技能

技能是代理 Claude Code 环境中可用的斜杠命令（如 `/stoa-audit`、`/deploy`）。它们从代理的工作目录中自动检测，并显示在设置面板中。

技能按**工作目录划分范围** — 当您使用特定工作目录创建房间时，只有该工作目录（项目/本地范围）的技能加上全局技能可用。这可以防止不同项目之间的技能冲突。

您可以在聊天中直接输入斜杠命令来调用技能：
```
/skill-name
```

## 工作区面板（文件编辑）

工作区面板是聊天右侧的代码查看器和文件浏览器。可以浏览、预览和编辑 AI 代理机器上的文件（包括远程服务器）。

### 打开面板

点击聊天头部的**面板切换按钮**（分栏图标）。面板在右侧打开，可通过拖动手柄调整大小。

### 文件树（Files 标签页）

**Files** 标签页显示房间工作目录的项目目录树。点击文件即可打开。文件夹点击展开/折叠。

### 代码查看器

文本文件以**语法高亮**（highlight.js）、**行号**和深色代码背景打开。顶部面包屑显示文件路径。

### 文件编辑

点击任何文本文件的 **Edit** 按钮进入编辑模式。使用 **CodeMirror 6** 提供语法高亮（JavaScript、TypeScript、Python、JSON、HTML、CSS、Markdown）。

- **保存**：`Ctrl+S`（`Cmd+S`）或点击保存按钮
- **全屏**：点击展开按钮使编辑器全宽显示（隐藏聊天面板）
- **冲突检测**：编辑时文件在磁盘上被修改，会显示重新加载或覆盖对话框
- **自动保存草稿**：未保存的编辑保存到浏览器，重新打开时显示恢复提示
- **键盘快捷键**：`Ctrl+/`（切换注释）、`Ctrl+Shift+D`（复制行）、`Alt+Shift+↑/↓`（复制行）、`Tab`（缩进）

如果 CodeMirror 加载失败（如无法访问 CDN），编辑器会回退到普通文本框。

### 文件操作（上下文菜单）

在文件树中右键点击文件或文件夹（平板电脑长按）打开上下文菜单：

- **新建文件** — 在选中文件夹中创建新文件
- **新建文件夹** — 创建新目录
- **重命名** — 重命名文件或文件夹
- **删除** — 删除文件（带确认对话框）

### Markdown 预览

`.md` 文件以带有标题、列表、代码块、表格和链接的格式化 Markdown 渲染。

### 图片预览

图片文件（PNG、JPG、GIF、WebP、SVG）以居中预览显示。对于远程代理，图片通过代理的 WebSocket 连接获取。

### Git 差异（Git 标签页）

**Git** 标签页显示未提交的更改（`git diff`），带有绿色/红色行高亮、文件头和更改统计。

### 可点击的文件路径

AI 代理在消息中提到文件路径（如 `/home/user/project/file.py`）时，路径会变为**可点击**——点击即可在工作区面板中打开文件。反引号代码块和代码围栏内的路径同样有效。

将鼠标悬停在文件树中的文件上会显示**下载按钮**（箭头图标）。点击即可下载到本地设备。远程代理的文件也通过 WebSocket 获取并作为浏览器下载传递。

### 远程文件浏览

工作区同时支持本地和远程代理。对于远程代理，文件操作通过代理的 WebSocket 连接代理——可以从任何设备（包括平板电脑和手机）浏览代理运行所在的任何机器上的文件。

## 自动化

Stoa 支持 **Slack 触发的自动化** — 当 Slack 事件符合条件时，自动向目标房间发送提示词的规则。

### 连接 Slack

进入**设置 > 自动化**，点击 **Connect Slack**。需要两个令牌：

- **App Token**（`xapp-1-...`）— 用于 WebSocket 连接（Socket Mode）
- **User Token**（`xoxp-...`）— 用于接收频道事件

详细步骤请参阅 [Slack 设置指南](doc-slack-setup)。

### 创建自动化规则

Slack 连接后，点击 **+ new rule** 创建规则：

- **名称** — 规则的描述性标签
- **触发事件** — 哪个 Slack 事件触发规则：`message`（公开频道）、`message.groups`（私有频道）或 `mention`
- **条件** — 可选过滤器：`message_text contains`、`message_text not_contains`、`message_text starts_with` 或 `matches_regex`。多个条件之间为 AND 关系
- **目标房间** — 接收提示词的 Stoa 房间
- **提示词模板** — 发送到房间的消息。可用变量：`{{slack_message_text}}`、`{{slack_message_link}}`、`{{slack_user}}`、`{{slack_channel}}`、`{{extracted_url}}`、`{{slack_thread_ts}}`

### 启用 / 禁用

每条规则都有一个启用/禁用开关。禁用的规则即使 Slack 事件匹配也不会触发。

## 架构概览

```
浏览器  <-->  WebSocket  <-->  server.js  <-->  代理 (stoa.js)
                                    |                   |
                                 SQLite DB      Claude Code CLI
                                                 或 Gemini CLI
                                                 或 Ollama
```

- **server.js** — HTTP + WebSocket 服务器，管理房间、消息和代理编排
- **public/** — 前端（无需构建步骤）
- **stoa.js** — 在每台代理机器上运行的代理客户端
- **claude-session.js** — 管理持久化的 Claude Code CLI 子进程
- **gemini-session.js** — 管理持久化的 Gemini CLI 子进程
- **gemini-adapter.js** — Gemini CLI 输出解析适配器
- **ollama-session.js** — 管理 Ollama API 调用（无需单独安装 CLI）
- **SQLite** — 所有数据本地存储在 `stoa.db`（使用 WAL 模式提升性能）

Stoa 支持多种 AI 后端。每个代理可以在添加时选择使用 **Claude Code CLI**、**Gemini CLI** 或 **Ollama**。所有后端都通过相同的代理客户端和编排层进行管理。Ollama 代理连接到本地 Ollama 服务器，无需单独安装 CLI。

**提示词中的房间上下文注入**：每当代理被触发时，服务器会将 `Room ID: <id>` 注入到代理的系统提示词中。这意味着代理始终知道自己在哪个房间中操作，无需显式传递 ID 即可调用 `sendProactiveMessage(roomId, ...)` 或执行其他感知房间的操作。

## 使用技巧

- **多代理，一个房间**：将互补的代理（如编码者和审查者）放在同一房间，通过 @提及让它们协作
- **专用房间**：为不同主题或项目创建单独的房间。每个房间维护自己的对话历史
- **工作目录**：为不同房间分配不同的工作目录，这样同一个代理可以在多个项目中工作
- **文件共享**：直接将文件拖放到聊天中 — 代理可以立即读取
- **先搜索**：在向代理提问之前，使用搜索功能检查是否已在其他房间中讨论过
